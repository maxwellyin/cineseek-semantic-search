Retrieval System

Search movies by meaning, not keywords.

CineSeek is a local-first movie search system built for portfolio-quality retrieval work. It combines a dual-tower ranker trained on real movie search judgments, FAISS serving for low-latency candidate retrieval, and an optional LangChain + Ollama layer for query rewrite, reranking, and explanation.

System Snapshot

Built to show retrieval engineering, not just a chat wrapper.

The core path stays explicit: sentence-transformer embeddings, a lightweight MLP dual tower, FAISS retrieval, and local agent reranking on top. The LLM layer enhances search quality without replacing the retrieval system.

What It Does

A retrieval stack you can actually explain in an interview.

Dual-Tower Retrieval

A trained query tower and item tower project movie search queries and movie metadata into the same retrieval space.

Cached Embeddings

Sentence-transformer outputs are cached locally, so retraining stays fast and focused on the retrieval head.

Agent Reranking

A local LangChain agent rewrites vague queries, reranks retrieved candidates, and explains why the top result fits.

Quick Run

Local-first setup

Train the retriever on MSRD, build a FAISS index, and optionally start a local Ollama model for the agent layer. 

source .venv/bin/activate
python -m flcr.train
env FLCR_DEVICE=cpu KMP_DUPLICATE_LIB_OK=TRUE python -m flcr.index
ollama serve
ollama pull qwen3:8b
uvicorn apps.demo.app:app --reload
