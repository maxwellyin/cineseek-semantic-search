What It Demonstrates

A retrieval-first search stack with an optional agent layer.

Dense Retrieval

Queries and movies are mapped into a shared representation space so results can match meaning, not just keyword overlap.

Candidate Search

FAISS provides low-latency nearest-neighbor retrieval over a movie corpus built from MovieLens and TMDB metadata.

LLM Reranking

A configurable agent layer can rewrite vague queries, rerank candidates, and summarize the final result list.

System Flow

What happens after you search

1. Retrieve

The query is embedded and matched against a FAISS index of movie representations.

2. Rerank

The agent reviews a broader candidate set, improves ordering, and filters obviously weak matches.

3. Explain

The final page includes a short summary so the result list feels inspectable rather than opaque.
